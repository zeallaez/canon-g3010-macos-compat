# Contributing

Issues and pull requests are welcome.

## Development

1. Use a Mac for package builds and hardware tests.
2. Run `make check`.
3. Keep Canon-owned files out of commits and release artifacts.
4. Document the macOS version, CPU architecture, printer firmware, transport,
   and observed CUPS/IPP status when reporting a hardware result.

## Pull requests

- Explain the user-visible change.
- Include the validation commands used.
- Update both English and Chinese documentation when behavior changes.
- Do not claim official Canon support.

By contributing, you agree that your original contribution is licensed under
the repository's MIT License.
